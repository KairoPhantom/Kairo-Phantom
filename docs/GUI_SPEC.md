# Kairo Phantom — GUI Specification

> Minimal thin Tauri overlay: drop a PDF → ask a question → see the answer with the source region highlighted, or see a clearly-labeled refusal.

## Design Principle

The Tauri overlay is **render-only**. There is no code path that displays ungrounded text. Every value shown in the UI must have a verified bounding-box anchor in the provenance store. If the grounding verifier cannot anchor a claim, the UI shows a refusal — never a guess.

## Screen Flow

### 1. Drop Zone (initial state)
- Full-window drag-and-drop area
- Label: "Drop a PDF here, or click to browse"
- Accepted formats: `.pdf`, `.txt`, `.md`, `.docx`
- On drop: document is ingested, chunks are created with bounding boxes, provenance chain is established

### 2. Question Input
- After ingestion, a text input appears: "Ask a question about this document"
- User types a natural-language question
- On submit: the question is sent to the Rust core, which routes through the Pack extraction + grounding cascade

### 3. Answer Panel (grounded answer)
- The answer text is displayed
- Below the answer: a **source region highlight** — the exact bounding-box region the answer came from, rendered as an overlay on the document image
- Click the highlight → scrolls/navigates to that region in the document view
- The answer is labeled with the grounding method used: `EXACT`, `FUZZY`, `SEMANTIC`, or `VISUAL`
- A "View source" button opens the document at the anchored page + bbox

### 4. Refusal Panel (no grounded source)
- When the grounding cascade returns `BLOCK`, the UI shows a **refusal** — not a blank panel, not a guess
- The refusal is clearly labeled and explained (see Refusal UX below)

## Refusal UX

When Kairo cannot ground an answer, it **stays silent**. The refusal is a first-class UI state, not an error or an empty result.

### Refusal rendering
```
┌─────────────────────────────────────────────┐
│  ⚠  No grounded source found               │
│                                              │
│  Kairo refuses to answer.                   │
│                                              │
│  Why? The grounding verifier could not      │
│  anchor this claim to any text region in    │
│  the document. The model may have produced  │
│  a plausible-sounding answer, but without   │
│  a verifiable source, Kairo will not show   │
│  it.                                        │
│                                              │
│  [Try a different question]                 │
└─────────────────────────────────────────────┘
```

### Refusal format function
The refusal state is generated by a deterministic function (`format_refusal`) that produces:
1. **Label:** "No grounded source found — refusing to answer"
2. **Indicator:** A visible warning icon (⚠) and a distinct background color
3. **Explanation:** A short "why?" string explaining which cascade stage blocked the answer

The refusal format function is tested in `tests/test_refusal_ui.py` to ensure it never renders a blank panel.

### Why? explanations by cascade stage
| Cascade stage that blocked | Explanation text |
|:---|:---|
| NORMALIZE failed | "The answer text could not be normalized for comparison." |
| EXACT failed | "No exact text match found in the document." |
| FUZZY < 0.92 | "No close match found (similarity below 0.92 threshold)." |
| SEMANTIC < 0.86 | "No semantic match found (similarity below 0.86 threshold)." |
| VISUAL IoU < 0.5 | "Visual region overlap too low (IoU below 0.5)." |
| BLOCK (all stages failed) | "The grounding verifier could not anchor this claim to any text region in the document." |

## UI Constraints

1. **No ungrounded text.** The overlay must never display a value without a verified anchor. This is enforced by the Rust core — the overlay only receives data that has passed the grounding verifier.
2. **No write-back.** v1 is READ + SUGGEST ONLY. The overlay has no "edit document" or "apply changes" button.
3. **Offline indicator.** A small badge shows "Offline" when the sidecar is running locally with no cloud connection.
4. **Pack indicator.** The active Pack (generic/invoice/paper/contract) is shown in the header.

│  │ Zone    │  │  - answer text       │  │
│  │         │  │  - source highlight  │  │

│  ├─────────┤  ├──────────────────────┤  │
│  │ Question│  │ Refusal Panel        │  │
│  │ Input   │  │  - label             │  │

         ↕ IPC (Tauri commands)
