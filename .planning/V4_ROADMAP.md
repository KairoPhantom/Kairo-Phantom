# Kairo Phantom v4.0 — Multimodal Vision Roadmap

## Vision Statement
Kairo Phantom v4.0 transforms from a text-only ghost-writer into a full **multimodal document AI peer** that can **see, read, and write** across every surface a human touches — documents, slides, code, images, canvases, and terminals.

---

## v4.0 Phase Breakdown

### Phase 7: Screen Vision & OCR Foundation
**Goal:** Capture screenshots of active windows as base64 context for vision-capable models.
- Integrate Windows `PrintWindow` / DXGIDuplicator for zero-flicker screenshots
- Pipe base64 frame to vision LLMs (LLaVA via Ollama, GPT-4o, Gemini Vision)
- Add `visual_context: Option<String>` to `DocumentContext`

### Phase 8: Image-Aware Agents
**Goal:** Vision-powered swarm agents that respond to what they *see*, not just what they *read*.
- `VisionDesignAgent`: Analyzes slide screenshots, suggests layout/copy improvements
- `VisionCodeAgent`: Reads terminal screenshots, diagnoses errors visually
- `VisionFormAgent`: Reads PDF/form images, fills in structured data

### Phase 8.5: Professional Formatting & Injection Bridges
**Goal:** Move beyond clipboard paste to high-fidelity, native document manipulation.
- Integrate `Adeu` MCP Server: Write Track Changes natively into DOCX without breaking formatting.
- Integrate `easy-notion-mcp`: True round-trip fidelity across 25 Markdown block types.
- Add `litchi` document creation path: Greenfield DOCX/PPTX/XLSX generation bypassing clipboard.
- Ship Kairo Figma Plugin: Native text injection via `figma-mcp-write-bridge` architecture.

### Phase 9: Cross-Platform Expansion
**Goal:** macOS (Accessibility API) + Linux (AT-SPI2) support
- Implement `MacOSAccessibilityReader` using `AXUIElement` APIs
- Implement `LinuxAtSpiReader` using AT-SPI2 D-Bus interface
- Port hotkey listener to `rdev` crate (cross-platform)

### Phase 10: Web App Fingerprinters (Browser Extension Mode)
**Goal:** Deep context from web apps without process-level hooks
- Chrome/Firefox extension that relays active page DOM/selection to Kairo HTTP API
- Smart fingerprinters for: Notion, Linear, GitHub, Google Docs, Jira, Confluence
- Plugin authors can ship browser extension + TOML combo

### Phase 11: Agent Memory & Learning
**Goal:** Kairo learns your writing style over sessions
- Rolling CRDT log becomes training data for fine-tune or RAG
- User-scoped memory store in `~/.kairo-phantom/memory/`
- `MemoryAgent`: Surfaces relevant past responses in system prompt

### Phase 12: Community Plugin Hub
**Goal:** Official plugin registry
- `kairo plugin install finance-specialist`
- Hosted manifest at `plugins.kairo.dev`
- Plugin signing with ed25519
